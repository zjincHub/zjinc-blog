每当聊到设计模式的时候，我脑袋里首先想到的是 java 这门语言。
联想到第一次学习工厂模式的疑惑和不解——明明一行代码就能实现，为什么要多写几个 class 呢？
随着不断地学习和项目经验的积累，我渐渐开始明白设计模式的重要性与必要性。
直到最近阅读《黑客与画家》，作者对设计模式的描述让我提起了深入学习模式的兴趣。

> 在面向对象编程的世界中，我们大量听到“模式”这个词，我觉得那些模式就你自己变成这个难题的人肉编译器。
> 当我在自己的程序中发现用到了模式，
> 这就表明某个地方出错了。程序的形式应该仅仅反映它所要解决的问题，代码中其他任何外加的形式都是一个信号，表明我对问题的抽象还不够深。
