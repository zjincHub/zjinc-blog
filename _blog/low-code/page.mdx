# 低代码时代背景

随着企业信息化、互联网化、以及数字化进程的不断深入，传统的前端组件化开发已无法满足人们对更高效率的追求。
2014年，市场研究公司 Forrester 首次提出了“低代码”的概念，并将其定义为：一种通过可视化拖拽和少量代码即可快速构建应用的开发方式，其核心目标是降低开发门槛，大幅提升软件交付效率。
自此，低代码开发平台迎来了爆发式增长，OutSystems、Mendix、Appian 等国际领先产品，以及宜搭、微搭、简道云等国内优秀平台纷纷涌现，为开发者和企业提供了更高效、更灵活的解决方案。

# 低代码平台分类

在深入学习低代码架构之前，我们先来了解一下低代码平台的常见类型及其特点：

1. 根据 **适用范围** 可分为

   _通用型低代码平台_：适用范围广，但需要更多配置，复杂度较高。

   _垂直型低代码平台_：专注于特定行业或业务场景，配置简单，但通用性较弱。

2. 根据 **是否商用** 可分为

   _职能型低代码平台_：专注于支持企业内部业务，与企业能力高度耦合，能够显著提升业务效率，但商业价值有限。

   _商用型低代码平台_：侧重于面向市场的商业化应用，灵活性更高，但可能对特定业务支持不足。

3. 根据 **技术实现** 可分为

   _编译型低代码平台_：用户完成设计后可输出一个独立运行的应用，开发成本较高，但运行灵活。

   _运行时型低代码平台_：输出的应用依赖于低代码平台环境运行，开发成本低，但独立性较差。

4. 根据 **使用方式** 可分为

   _模板型低代码平台_：提供丰富的模板库，用户仅需选择和修改模板即可完成开发，适合快速搭建。

   _组件型低代码平台_：提供模块化组件库，用户通过拖拽组件完成构建，适用于动态交互应用。

   _代码增强型低代码平台_：结合可视化开发与手写代码，适合实现复杂业务逻辑。

当然，低代码平台还有许多其他的分类维度，这里不再一一列举。了解这些分类的意义在于：**在构建低代码平台时，需要明确目标和方向，避免后续开发中的弯路**。 清晰的分类有助于我们设计更高效、更符合需求的低代码架构。
